# ABLITERATED.cloud website v0.8.0

This release publishes the complete model field-notes blog and sharpens the
site's brain identity without starting any model endpoint or GPU.

## Four exact model stories

- `huihui-ai/Huihui-Qwen3.6-35B-A3B-abliterated`
- `YuYu1015/YuYu1015-Ornith-1.0-35B-abliterated`
- `huihui-ai/Huihui-Qwythos-9B-Claude-Mythos-5-1M-abliterated`
- `cebeuq/Ornith-1.0-397B-abliterated-W4A16`

Each article is available as semantic HTML and Markdown. Together they cover
architecture, parameter and storage figures, model history, post-training,
abliteration, quantization, evaluations, deployment implications and the
publicly verifiable people or publishers behind the derivative. Claims are
linked to primary sources, publisher measurements remain labeled, and upstream
benchmarks are not attributed to untested derivative weights.

The blog also ships Blog and TechArticle structured data, canonical metadata,
RSS, sitemap entries and agent-readable discovery. The prepared edge Worker
preserves canonical directory slashes so every blog URL resolves directly
without a redirect loop.

## Brain identity

The interactive hero now uses 1,280 deterministic nodes, denser connections
and visible cortical folds. The fade is gone, and a brain-and-synapse mark now
serves as the navigation logo and favicon across SVG, PNG and ICO formats.

The supporting statement is now exactly:

> Abliterated open models. One simple API.

## Runtime safety

This is a static website release. It does not import Modal runtime definitions,
deploy an endpoint, wake a model or start a GPU. Ornith 397B remains visible in
the catalog and deployment-disabled.
